What is this?

It automatically collects subscription pricing (USD amounts and per-tier features) for multiple companies from their official US pricing pages. It runs weekly, detects price/tier/feature changes vs. the previous week, and shows everything on this dashboard.

Screens

Overview Latest price table per company × tier (monthly/annual/currency/billing). A badge shows changes this week.
Company detail Click a company name. Shows a price history chart, per-tier features/limits, and source links.
Changes All detected changes (old → new, one-line summary), filterable by company.
Runs Recent run times and per-company success/failure/confidence. [Run now] triggers an immediate collection. You can also clear logs / reset all data here.
Compare Companies Pick companies to compare side by side: a price scatter, per-company paid features by category, a category matrix, and a heuristic value score. Optional AI categorization groups features dynamically. Click [Save this comparison] to keep the result as a card frozen at save time — reopen it anytime without re-running.

Adding / removing companies

Manage companies on the Companies screen. The list is stored in the DB and survives redeploys/migrations, working the same on Render and locally. After adding, click Runs → [Run now].

Multiple sources per company + priority

Some companies list pricing only in app stores. Add multiple sources per company; each is tracked independently and the primary source is chosen by this priority:

Official site > Google search > App Store > Play Store

  • The Overview shows the highest-priority source with data as primary; others appear under "see prices".
  • Google search: leave the URL blank to auto-build google.com/search?q=Company pricing plans free trial (US/English).
  • App Store: the URL needs /us/ for USD (auto-corrected if omitted).
  • Play Store: &gl=us&hl=en is added automatically.
  • Stores/search may be bot-blocked or localized; non-USD results are flagged as low confidence.

Feature comparison (free vs paid)

The Company detail screen shows a per-source feature matrix — features as rows, tiers as columns, with ✓/— so free-vs-paid differences are clear (free columns blue, paid green).

How to collect

  • Web: Runs → [Run now] (runs in background)
  • Automatic (weekly): Render fires every Mon 09:00 ET; locally the OS scheduler handles it
  • CLI: python -m app.core.pipeline (collect/store/diff without the web)

Reading the badges

Free tier a $0 tier exists  ·  Free trial a trial is offered  ·  Needs review low confidence  ·  KRW non-USD currency (never auto-converted)

Prices (monthly / annual-per-month) are per-seat, per-month USD. "Private" means Contact-Sales (not an error — price left blank with a note).

Notes (dev/ops)

  • Internal API: GET /api/snapshots/latest, GET /api/company/<name>/history, POST /run-now
  • Health: GET /healthz
  • All data is one SQLite file — migrating environments is just a file copy.

이 서비스는 무엇인가요?

여러 업체의 구독 Pricing 정보(USD 금액 · Tier별 기능)를 공식 US Pricing 페이지 기준으로 자동 수집합니다. 주 1회 실행되며, 이전 주 대비 가격·티어·기능 변동을 감지해 이 대시보드로 보여줍니다.

화면 안내

현황 전체 업체 × 티어의 최신 가격표(월/연 환산/통화/과금단위). 이번 주 변동이 있으면 배지로 표시됩니다.
업체 상세 현황에서 업체 이름을 클릭하면 이동. 가격 추이 차트와 현재 티어별 기능·limits, 원본 페이지 링크를 봅니다.
변동 로그 감지된 모든 변동 이력(old → new, 한 줄 요약). 업체별로 필터할 수 있습니다.
수집 상태 최근 실행 시각과 업체별 성공/실패/신뢰도. [지금 수집] 버튼으로 즉시 1회 수집을 돌릴 수 있습니다.
업체 비교 여러 업체를 골라 나란히 비교합니다: 가격 산점도 · 업체별 유료 기능(카테고리별) · 카테고리 매트릭스 · 가치 점수. AI 카테고리 분류로 기능을 자동 그룹화할 수 있고, [이 비교 저장]을 누르면 결과가 저장 시점 그대로 카드로 보관되어 다시 비교하지 않고 언제든 다시 열어볼 수 있습니다.

업체 추가/삭제

업체 관리 화면에서 웹으로 직접 추가/삭제합니다. 목록은 DB에 저장되어 재배포·환경 이전에도 유지되며, Render·로컬 어느 환경에서나 동일하게 동작합니다. 추가 후 수집 상태 → [지금 수집]을 누르면 새 업체가 수집됩니다.

  • 업체명 (필수) — 데이터의 키로 사용
  • 소스 종류 — 웹 / App Store / Play Store / 기타
  • 소스 URL (필수) — 해당 소스의 가격 페이지 URL

한 업체에 여러 소스 + 우선순위

가격이 웹에 없고 스토어에만 있는 업체도 있습니다. 업체마다 여러 소스를 등록하면 각 소스를 독립 수집·추적하고, 다음 우선순위로 대표 출처를 정합니다:

공식 홈페이지 > 구글 검색 > App Store > Play Store

  • 현황에는 우선순위가 가장 높은(USD·신뢰도 정상) 출처가 대표로 표시되고, 나머지는 "추가 출처"로 명시됩니다.
  • 구글 검색: URL을 비우면 google.com/search?q=업체명 pricing 을 US·영어로 자동 생성합니다.
  • App Store: apps.apple.com/us/app/idXXXXXXXX — /us/ 가 USD 핵심(빠뜨려도 자동 보정)
  • Play Store: play.google.com/store/apps/details?id=...&gl=us&hl=en (자동 추가)
  • 스토어/검색은 봇 차단·지역화가 있어, 결과가 비-USD면 신뢰도 낮음으로 표시됩니다.

티어별 기능 비교 (무료 vs 유료)

업체 상세 화면에 출처별로 기능 비교표가 나옵니다. 전체 기능을 행으로, 각 티어를 열로 두고 보유 여부(✓/—)를 표시해 무료 티어와 유료 티어의 기능 차이를 한눈에 비교할 수 있습니다(무료 열은 파랑, 유료 열은 초록).

삭제해도 과거 수집 이력(스냅샷·변동 로그)은 보존됩니다. 최초 실행 시 companies.yaml 의 목록이 DB로 1회 시드된 뒤로는 DB가 기준이 됩니다.

수집 실행 방법

  • 웹에서: 수집 상태 → [지금 수집] (백그라운드 실행)
  • 자동(주 1회): Render는 매주 월 09:00 ET 자동 발화 / 로컬 PC는 OS 작업 스케줄러가 담당
  • 명령줄: python -m app.core.pipeline (웹 없이도 수집·저장·변동감지 완결)

표시 항목 읽는 법

배지

변동 N 이번 주 감지된 변동 수  ·  검수 필요 신뢰도 낮음(low) — 사람이 확인 권장  ·  KRW USD가 아닌 통화 감지(임의 환산하지 않음)

변동 유형

price_changed 가격 변경   tier_added 티어 신설   tier_removed 티어 삭제   feature_changed 기능 변경

가격 표기

  • 월 / 연(월환산) — 모두 1인·1개월 기준 USD 금액입니다.
  • 비공개 — Enterprise처럼 "Contact Sales"인 경우. 에러가 아니며 가격은 비워두고 사유를 메모로 남깁니다.

US / USD 기준

모든 가격은 USD 기준입니다. 수집 시 en-US 로케일을 강제하지만, 로컬 PC(한국 IP)에서는 일부 사이트가 비-USD를 줄 수 있습니다. 이때 결과를 자동으로 신뢰도 낮음(low) 으로 표시하고 임의 환산은 하지 않습니다. 해당 업체의 pricing_url을 US 전용 경로(?currency=usd, /us/pricing 등)로 지정하면 개선됩니다.

참고 (개발/운영)

  • 내부 API: GET /api/snapshots/latest, GET /api/company/<name>/history, POST /run-now
  • 상태 점검: GET /healthz
  • 데이터는 SQLite 파일 1개. 환경 이전은 이 파일 복사로 끝납니다.
