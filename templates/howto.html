사용법 (How-to)

이 서비스는 무엇인가요?

여러 업체의 구독 Pricing 정보(USD 금액 · Tier별 기능)를 공식 US Pricing 페이지 기준으로 자동 수집합니다. 주 1회 실행되며, 이전 주 대비 가격·티어·기능 변동을 감지해 이 대시보드로 보여줍니다.

화면 안내

현황 전체 업체 × 티어의 최신 가격표(월/연 환산/통화/과금단위). 이번 주 변동이 있으면 배지로 표시됩니다.
업체 상세 현황에서 업체 이름을 클릭하면 이동. 가격 추이 차트와 현재 티어별 기능·limits, 원본 페이지 링크를 봅니다.
변동 로그 감지된 모든 변동 이력(old → new, 한 줄 요약). 업체별로 필터할 수 있습니다.
수집 상태 최근 실행 시각과 업체별 성공/실패/신뢰도. [지금 수집] 버튼으로 즉시 1회 수집을 돌릴 수 있습니다.

업체 추가/삭제

업체 관리 화면에서 웹으로 직접 추가/삭제합니다. 목록은 DB에 저장되어 재배포·환경 이전에도 유지되며, Render·로컬 어느 환경에서나 동일하게 동작합니다. 추가 후 수집 상태 → [지금 수집]을 누르면 새 업체가 수집됩니다.

  • 업체명 (필수) — 데이터의 키로 사용
  • Pricing URL (권장) — US Pricing 페이지 URL. 가장 안정적입니다.
  • Homepage (선택) — 미지정 시 {homepage}/pricing 으로 추정

삭제해도 과거 수집 이력(스냅샷·변동 로그)은 보존됩니다. 최초 실행 시 companies.yaml 의 목록이 DB로 1회 시드된 뒤로는 DB가 기준이 됩니다.

수집 실행 방법

  • 웹에서: 수집 상태 → [지금 수집] (백그라운드 실행)
  • 자동(주 1회): Render는 매주 월 09:00 ET 자동 발화 / 로컬 PC는 OS 작업 스케줄러가 담당
  • 명령줄: python -m app.core.pipeline (웹 없이도 수집·저장·변동감지 완결)

표시 항목 읽는 법

배지

변동 N 이번 주 감지된 변동 수  ·  검수 필요 신뢰도 낮음(low) — 사람이 확인 권장  ·  KRW USD가 아닌 통화 감지(임의 환산하지 않음)

변동 유형

price_changed 가격 변경   tier_added 티어 신설   tier_removed 티어 삭제   feature_changed 기능 변경

가격 표기

  • 월 / 연(월환산) — 모두 1인·1개월 기준 USD 금액입니다.
  • 비공개 — Enterprise처럼 "Contact Sales"인 경우. 에러가 아니며 가격은 비워두고 사유를 메모로 남깁니다.

US / USD 기준

모든 가격은 USD 기준입니다. 수집 시 en-US 로케일을 강제하지만, 로컬 PC(한국 IP)에서는 일부 사이트가 비-USD를 줄 수 있습니다. 이때 결과를 자동으로 신뢰도 낮음(low) 으로 표시하고 임의 환산은 하지 않습니다. 해당 업체의 pricing_url을 US 전용 경로(?currency=usd, /us/pricing 등)로 지정하면 개선됩니다.

참고 (개발/운영)

  • 내부 API: GET /api/snapshots/latest, GET /api/company/<name>/history, POST /run-now
  • 상태 점검: GET /healthz
  • 데이터는 SQLite 파일 1개. 환경 이전은 이 파일 복사로 끝납니다.
